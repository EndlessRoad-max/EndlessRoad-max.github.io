三、 深度深度原理解析

1. 为什么必须使用“继承”？

在 Java 中，绘图不是“主动命令式”的，而是“被动响应式”的。

  • 控制权翻转：你无法决定“现在画”，你只能决定“当系统要求画的时候，画什么”。
  • 模板方法模式：通过继承 JPanel 并重写 paint，你实际上是在给系统的“绘图流程”填空。

2. 什么是“钩子函数” (Hook Function)？

paint(Graphics g) 就是一个钩子。

  • 系统在运行过程中，在需要刷新的时刻（如窗口缩放、被遮挡后重新显示）预留了一个位置。
  • 你的重写操作，就是把你的代码“钩”在了系统的自动运行链条上。

3. 监控器：事件派发线程 (EDT)

Java 并不是时刻盯着你的代码，而是有一个后台循环：Event Dispatch Thread (EDT)。

  • Step 1: 操作系统检测到窗口变化（Resize/Move）。
  • Step 2: 操作系统发消息给 JVM 事件队列。
  • Step 3: EDT 线程取出消息，发现是“重绘”需求。
  • Step 4: EDT 自动调用你的 paint(g) 方法，并将 Graphics 画笔交给你使用。

四、 重点避坑指南

  1. 坐标系统：左上角是 $(0, 0)$。$X$ 轴向右递增，$Y$ 轴向下递增。
  2. 刷新残影：如果不写 super.paint(g)，窗口在缩放时可能会出现画面重叠或背景花屏。
  3. 性能注意：不要在 paint 方法里创建对象（如 new Font），因为该方法会被频繁调用，频繁创建对象会导致内存压力大，建议将对象定义为类成员变量。
