Java 路径体系最终版笔记

🧩 一、核心结论（总纲）

Java 中的路径体系主要分为两类：
一类是基于文件系统的路径，用于 File 和 IO 流操作；
另一类是类路径（Classpath），用于 JVM 加载类和资源。
两者来源不同、用途不同，属于两套独立体系，不能混淆。

🏁 八、最终总结（面试背诵版）

Java 中的路径体系主要分为两类：一类是基于文件系统的路径，主要用于 File 和 IO 流操作，其中相对路径基于程序运行时的工作目录（Working Directory）解析，绝对路径则直接定位到操作系统中的真实文件位置；另一类是类路径（Classpath），用于 JVM 加载类和资源，资源加载方法通常基于类路径进行解析。需要注意，不同 API 的路径解析基准不同，文件系统路径与类路径属于两套独立体系，不能混淆。

🚀 九、补充一句（面试加分）

在实际开发中，必须明确当前 API 使用的是文件系统路径还是类路径，否则容易导致资源加载失败的问题。
