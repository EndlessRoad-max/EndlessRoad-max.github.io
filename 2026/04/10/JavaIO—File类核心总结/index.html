JavaIO—File类核心总结

📘 Java IO 笔记 —— File 类核心总结

🧩 一、File 类本质

在 Java SE 中：

File 类本质是对文件系统路径的抽象表示

✅ 核心特点

  • ✔ 本质是一个路径对象（String 封装）
  • ✔ 可以表示：
    • 文件 📄
    • 目录 📁
  • ❗ 不关心是否真实存在

此时：

  • ❌ 文件不一定存在
  • ✅ 只是表示“这个路径”

🧠 二、File 的核心能力

1️⃣ 文件/目录操作（元数据操作）

2️⃣ 目录操作

3️⃣ 信息查询

⚠️ 三、重要限制（高频考点）

❌ File 不能操作文件内容

  • 不能读 ❌
  • 不能写 ❌

🔥 四、路径 vs 类型（核心理解）

❗ File 对象本身不区分文件 or 目录

👉 这个路径：

  • 可以是文件
  • 也可以是目录

✅ 类型由“方法决定”

方法 结果
createNewFile() 创建文件 📄
mkdir / mkdirs 创建目录 📁

1️⃣ 创建文件

2️⃣ 创建目录

✔ 可以递归创建目录（mkdirs）

🏁 最终总结（面试版）

File 类是 Java 中用于表示文件系统路径的对象，它封装了路径信息，并提供对文件或目录进行创建、删除和查询等操作的方法。但 File 不负责文件内容的读写，文件内容操作需要通过 IO 流完成。此外，Java 中创建文件时必须保证父目录存在，因此通常需要先创建目录再创建文件。
