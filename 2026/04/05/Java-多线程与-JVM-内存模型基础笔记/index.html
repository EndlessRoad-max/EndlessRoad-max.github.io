Java 多线程与 JVM 内存模型基础笔记

知识点笔记：Java 多线程与 JVM 内存模型基础

1. 线程间资源共享的本质

1.1 操作系统视角：进程与线程

  • 进程 (Process)： 操作系统分配资源的基本单位。进程间的资源互相隔离（拥有独立的虚拟地址空间）。
  • 线程 (Thread)： CPU 调度的基本单位，被称为“轻量级进程”。同一个进程下的多个线程共享该进程的内存地址空间。
  • 优势： 相比于进程间通信（IPC），线程间共享数据更加方便，上下文切换开销更小（无需切换全局页表等）。

1.2 Java 代码表现：共享并非“默认发生”

虽然同进程内的线程拥有访问同一片内存空间的能力，但在实际代码中，资源是否真正被“共享”，取决于对象的分配方式：

  • 继承 Thread 类 的方式： * 每次 new MyThread() 都会在堆中创建一个新的独立对象。
    • 线程各自操作自己实例中的成员变量，资源默认不共享。
    • 若要共享，需将变量声明为 static（类变量，存放在方法区）。
  • 实现 Runnable 接口 的方式： * 可以只 new 一个 Runnable 实现类实例，然后将这同一个实例传入多个 Thread 中运行（new Thread(runnable)）。
    • 多个线程共同访问这同一个实例的成员变量，天然实现资源共享。

核心结论： 所谓的“线程间资源共享”，是指它们都有资格访问进程分配的那片内存；但具体是否共享，需要通过代码设计（传递同一个对象引用，或使用静态变量）来维系。

2. JVM 运行时数据区（内存划分）

在 Java 虚拟机中，内存被严格划分为“线程共享”和“线程私有”两大区域。这直接解释了为什么有些数据会相互影响，而有些数据相互独立。

2.1 线程共享区域（所有线程共用，容易引发并发问题）

  • 堆 (Heap)： *存储内容： 几乎所有的对象实例（new 出来的对象）和数组。
    • 特点： JVM 中最大的一块内存。只要线程拿到了堆中对象的引用，就能对其进行修改。Runnable 方式的共享对象即存在于此。
  • 方法区 (Method Area / Metaspace)： 存储内容： 已被 JVM 加载的类结构信息、常量、*静态变量 (Static Variables)、即时编译器编译后的代码等。
    • 特点： 这解释了为什么设置 static 修饰的变量可以被所有线程共享。

2.2 线程私有区域（每个线程独立拥有一份，天然线程安全）

  • Java 虚拟机栈 (JVM Stack)： *工作机制： 线程每次调用一个方法，就会压入一个“栈帧”(Stack Frame)。方法结束则栈帧出栈。
    • 存储内容： 局部变量表（基本数据类型、对象引用指针）、操作数栈、动态链接、方法出口等。
    • 特点： 即使多个线程执行同一段代码，方法内部的局部变量也是互相隔离的。
  • 程序计数器 (Program Counter Register)： *作用： 记录当前线程执行到了哪一条字节码指令。
    • 特点： CPU 轮转切换线程时，通过它来记住当前线程挂起前的位置，以便恢复执行。内存占用极小。
  • 本地方法栈 (Native Method Stack)： *作用： 为 JVM 调用底层 Native 方法（如 C/C++ 编写的底层系统接口）服务。与虚拟机栈功能类似。

3. 进阶思考：线程安全问题的前置概念

正是因为多个线程共享了 堆 和 方法区，当多个线程同时对这部分共享内存进行“读写”操作时，就会产生资源竞争，导致数据不一致（例如：经典的卖票超卖问题）。

  • 下一步学习方向： 了解 Java 是如何通过同步机制（如 synchronized 关键字、Lock 接口、volatile 关键字等）来保证多线程环境下共享数据的安全性的。
